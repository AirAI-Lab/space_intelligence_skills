## 1. 系统概述

### 1.1 应用场景

园区监测系统用于工业园区、商业园区、住宅小区等场景的智能安全管理，实现：
- 周界安全防护
- 人员行为监控
- 车辆管理
- 异常事件检测

### 1.2 核心方案

| 模型 | 任务 | 类别 | 数据集 | FPS | 位置 |
|------|------|------|--------|-----|------|
| **YOLOv8-Pose** | 检测+行为 | 12类 | 4500张 | 30+ | 边缘 |
| **YOLOv8-Obb** | 停车位检测 | 1类 | 500张 | 25+ | 边缘 |

### 1.3 分类标准

| 判断依据 | YOLOv8-Pose | YOLOv8-Obb |
|---------|------------|-----------|
| **有固定形状？** | ✅ 是 | ✅ 是（旋转框） |
| **边界清晰？** | ✅ 是 | ✅ 是 |
| **核心特征** | 人体/车辆形状 | 停车位边界 |
| **样本需求** | 200-1000张/类 | 500张 |

### 1.4 项目结构

```
park_monitoring/
├── configs/
│   └── park_monitoring.yaml      # 统一配置
│
├── models/
│   ├── yolo/                      # YOLO 检测器
│   └── unified.py                 # 统一接口
│
├── scripts/                       # 工具脚本
├── deployment/                    # 部署代码
└── docs/
    └── MANUAL.md                  # 本文档
```

## 3. 技术方案

### 3.1 YOLOv8-Pose 架构

```
输入图像 (640×640)
    │
    ├─ Backbone (CSPDarknet)
    ├─ Neck (PANet)
    ├─ Head (Detect) → 12类检测
    └─ Pose Head → 关键点（用于行为分析）
```

### 3.2 YOLOv8-Obb 架构

```
输入图像 (640×640)
    │
    ├─ Backbone (CSPDarknet)
    ├─ Neck (PANet)
    └─ Head (Obb) → 旋转框检测
```

### 3.3 边缘部署架构

```
园区摄像头
    │
    ├─ 边缘设备 (Jetson)
    │    ├─ YOLOv8-Pose (30 FPS)
    │    │    ├─ 周界安全（4类）
    │    │    ├─ 人员行为（5类）
    │    │    └─ 车辆管理（3类）
    │    │
    │    ├─ YOLOv8-Obb (25 FPS)
    │    │    └─ 停车位检测（1类）
    │    │
    │    └─ 本地报警
    │
    └─ 云端
         ├─ 数据存储
         ├─ 轨迹分析
         └─ 报表生成
```

## 8. 常见问题

### Q1: 与水利巡检的区别？

| 对比项 | 水利巡检 | 园区监测 |
|--------|---------|---------|
| **场景** | 开放水域 | 封闭园区 |
| **模型数** | 2个（YOLOv8 + DINOv3） | 2个（YOLOv8-Pose + Obb） |
| **云端需求** | 需要（水质分割） | 可选（仅存储） |
| **实时性** | 边缘实时 | 边缘实时 |

### Q2: 为什么需要 YOLOv8-Obb？

**原因**：
- 停车位有不同角度
- 普通边界框无法准确描述
- 旋转框可以精确检测停车位边界

### Q3: 停车位如何判断占用？

**方法**：
1. 检测停车位边界（Obb）
2. 检测车辆（普通检测）
3. 判断车辆与停车位的重叠
4. 重叠率 > 50% → 占用
