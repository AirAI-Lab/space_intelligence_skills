## 1. YOLOv8 检测任务（11类）

### 任务 1: person（人员检测）

| 属性 | 内容 |
|------|------|
| **类别ID** | 0 |
| **英文** | person |
| **中文** | 人员 |

#### 定义

**什么情况算作检测到**：
- 画面中出现人体躯干（头+上身）
- 最小高度 ≥ 50 像素
- 可见度 ≥ 30%（允许部分遮挡）

**不算作检测的情况**：
- 人体高度 < 50 像素
- 可见度 < 30%
- 背影/倒影

#### 标注规范

```
标注格式: 0 x_center y_center width height

标注要点:
- 边界框紧贴人体轮廓
- 包含完整躯干（头到脚/腰）
- 遮挡时标注可见部分
- 多人时分别标注

示例:
0 0.5 0.6 0.15 0.35  # 正常站立
0 0.3 0.7 0.12 0.25  # 部分遮挡
```

#### 技术方案

| 项目 | 内容 |
|------|------|
| **模型** | YOLOv8-Pose |
| **位置** | 边缘（Jetson） |
| **FPS** | 30+ |

#### 难点和挑战

1. **小目标**：远景人员 < 50px
2. **遮挡**：植被、建筑物遮挡
3. **姿态变化**：蹲下、躺卧
4. **光照**：逆光、阴影

#### 数据集来源建议

| 来源 | 数量 | 说明 |
|------|------|------|
| COCO | 1000+ | 直接使用 train2017 子集 |
| 现场采集 | 300+ | 无人机视角、岸边场景 |
| **总计** | **1300** | |

#### 评估指标

| 指标 | 目标值 |
|------|--------|
| Precision | ≥ 0.90 |
| Recall | ≥ 0.85 |
| mAP@50 | ≥ 0.90 |

#### 报警规则

```
触发条件: 无（基础类别，不直接报警）
用途: 作为其他行为检测的前置条件
```

#### 部署优先级

| 优先级 | 级别 |
|--------|------|
| **P0** | 最高（基础类别） |

#### 后处理需求

| 需求 | 说明 |
|------|------|
| ID 追踪 | ByteTrack |
| 轨迹记录 | 用于行为分析 |

### 任务 2: fishing_person（违规钓鱼）

| 属性 | 内容 |
|------|------|
| **类别ID** | 1 |
| **英文** | fishing_person |
| **中文** | 钓鱼人员 |

#### 定义

**什么情况算作检测到**：
- 人员手持鱼竿（细长物体，长度 > 身高 0.5倍）
- 静态姿态持续时间 ≥ 30 秒
- 位于岸边/台阶/护栏旁

**不算作检测的情况**：
- 手持其他长物体（扫帚、棍子）
- 动态移动中
- 持续时间 < 30 秒

#### 标注规范

```
标注格式: 1 x_center y_center width height

标注要点:
- 边界框包含人员 + 鱼竿
- 鱼竿需要清晰可见
- 标注时标记 fishing_person 类别

示例:
1 0.45 0.6 0.3 0.4   # 坐在岸边钓鱼
1 0.3 0.5 0.25 0.35  # 站立钓鱼
```

#### 技术方案

| 项目 | 内容 |
|------|------|
| **模型** | YOLOv8-Pose |
| **位置** | 边缘（Jetson） |
| **FPS** | 30+ |
| **后处理** | 静止检测 + ID 追踪 |

#### 难点和挑战

1. **鱼竿细长**：小目标检测
2. **遮挡**：鱼竿被身体遮挡
3. **姿态多样**：坐、站、蹲
4. **误报**：其他长物体

#### 数据集来源建议

| 来源 | 数量 | 说明 |
|------|------|------|
| 现场采集 | 500+ | 不同角度、光照 |
| 网络爬取 | 150+ | YouTube 钓鱼视频 |
| **总计** | **650** | |

#### 评估指标

| 指标 | 目标值 |
|------|--------|
| Precision | ≥ 0.85 |
| Recall | ≥ 0.80 |
| mAP@50 | ≥ 0.85 |

#### 报警规则

```
触发条件: 
  - 检测到 fishing_person
  - 持续时间 ≥ 30 秒
  - 置信度 ≥ 0.6

报警级别: 警告
推送方式: 现场语音 + APP 通知
```

#### 部署优先级

| 优先级 | 级别 |
|--------|------|
| **P0** | 最高 |

#### 后处理需求

| 需求 | 说明 |
|------|------|
| 静止检测 | 连续帧位置变化 < 阈值 |
| ID 追踪 | 记录持续时间 |
| 证据截图 | 报警时保存关键帧 |

### 任务 3: swimming_person（违规游泳）

| 属性 | 内容 |
|------|------|
| **类别ID** | 2 |
| **英文** | swimming_person |
| **中文** | 游泳人员 |

#### 定义

**什么情况算作检测到**：
- 人员在水面区域（需定义 ROI）
- 有肢体动作（手臂/腿部运动）
- 头部露出水面

**不算作检测的情况**：
- 在岸边（未进入水面）
- 静态浮漂
- 船上人员

#### 标注规范

```
标注格式: 2 x_center y_center width height

标注要点:
- 边界框包含人员 + 水面涟漪
- 只标注在水面区域的人员
- 多人分别标注

示例:
2 0.5 0.7 0.2 0.25   # 游泳中
2 0.4 0.65 0.18 0.22 # 浮在水面
```

#### 技术方案

| 项目 | 内容 |
|------|------|
| **模型** | YOLOv8-Pose |
| **位置** | 边缘（Jetson） |
| **FPS** | 30+ |
| **后处理** | ROI 判断 + 姿态分析 |

#### 难点和挑战

1. **水面反光**：干扰检测
2. **头部小**：远景时 < 30px
3. **姿态变化**：蛙泳、自由泳
4. **波浪遮挡**：部分身体在水下

#### 数据集来源建议

#### 报警规则

```
标注格式: 4 x_center y_center width height

- 标注越界的人员
- 需要配置警戒线坐标
```

1. **警戒线定义**：不同场景不同

- 表面严重锈蚀

**不算作检测的情况**：

- 正常磨损

```
标注格式: 8 x_center y_center width height

标注要点:
- 边界框包含破损管道
- 标注时描述破损类型
- 破损区域也要包含

```

#### 技术方案

| 项目 | 内容 |
|------|------|
| **模型** | YOLOv8 |
| **位置** | 边缘（Jetson） |
| **FPS** | 30+ |

#### 难点和挑战

1. **样本极稀缺**：破损事件很少

## 3. 评估指标定义

### 3.1 检测指标（YOLOv8）

| 指标 | 定义 | 计算方式 |
|------|------|---------|
| **mAP@50** | IOU=0.5 时的平均精度 | 所有类别的 AP 平均 |
| **mAP@50:95** | IOU=0.5-0.95 时的平均精度 | 所有 IOU 阈值的 AP 平均 |
| **Precision** | 精确率 | TP / (TP + FP) |
| **Recall** | 召回率 | TP / (TP + FN) |
| **F1** | F1 分数 | 2 * (P * R) / (P + R) |

### 3.2 分割指标（DINOv3+SAM3）

| 指标 | 定义 | 计算方式 |
|------|------|---------|
| **mIoU** | 平均交并比 | 所有类别的 IoU 平均 |
| **Boundary F1** | 边界 F1 分数 | 边界像素的 F1 |
| **Pixel Accuracy** | 像素准确率 | 正确像素 / 总像素 |

### 3.3 系统指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **FPS** | 每秒帧数 | ≥ 30 (边缘) |
| **Latency** | 单帧延迟 | < 50ms |
| **Memory** | GPU 内存占用 | < 8GB |
| **Power** | 功耗 | < 25W |
