## 1. 系统概述

### 1.1 应用场景

施工安全监测系统用于建筑工地、施工现场等场景的智能安全管理，实现：
- 人员安全装备检测
- 施工区域管控
- 危险行为识别
- 环境安全监测

### 1.2 核心方案

| 模型 | 任务 | 类别 | 数据集 | FPS | 位置 |
|------|------|------|--------|-----|------|
| **YOLOv8-Pose** | 人员检测+行为 | 15类 | 5500张 | 30+ | 边缘 |
| **C-RADIOv4-H + SigLIP2-g** | 场景零样本检测 | 4类 | 零样本 | 5-10 | 云端 |

### 1.3 特殊说明

**为什么施工安全需要 RADIO 零样本？**

| 检测方式 | YOLOv8-Pose | C-RADIOv4 |
|---------|------------|-----------|
| **适用目标** | 固定形状、边界清晰 | 无固定形状、渐变边界 |
| **样本需求** | 200-1000张/类 | **零样本**（文本提示） |
| **典型任务** | 安全帽、反光衣、机械 | 裸土、扬尘、坑内积水 |

**RADIO 零样本检测场景**（样本稀缺、无固定形状、实时性要求低）：

| 类别 | 英文 | 特征 | 报警级别 | F1 |
|------|------|------|----------|-----|
| 裸土未覆盖 | bare_soil_uncovered | 黄褐色泥土无覆盖 | warning | 83.67% |
| 扬尘污染 | dust_pollution | 灰白色尘雾弥漫 | warning | 76.00% |
| 坑内积水 | pit_water_accumulation | 基坑底部浑浊积水 | info | 73.47% |
| 基坑边材料堆放 | material_near_pit | 基坑边缘堆放材料 | warning | 待验证 |

> 总体验证准确率: **77.70%** (148张, 阈值0.25, 最小面积0.003)

## 3. 技术方案

### 3.1 YOLOv8-Pose 架构

```
输入图像 (640×640)
    │
    ├─ Backbone (CSPDarknet)
    ├─ Neck (PANet)
    ├─ Head (Detect) → 15类检测
    └─ Pose Head → 关键点（用于行为分析）
```

### 3.2 C-RADIOv4 零样本检测架构（云端）

```
输入图像 (896×896)
    │
    ├─ C-RADIOv4 ViT-H Backbone → 3136 patch features
    ├─ SigLIP2-g 文本编码 → 裸土/扬尘/积水/材料堆放 提示
    └─ Patch similarity → 检测热力图 → 区域分割
```

**提示词设计策略**：
- 裸土: 强调干燥泥土纹理 + 无覆盖否定语义 + 排除水面/湿泥的负例
- 扬尘: 强调"空气中尘雾"的空间特征，区分自然雾气
- 坑内积水: 强调水面反射/镜面光泽特征（非位置约束），排除干燥裸土
- 材料堆放: 强调"在基坑边缘"的空间关系

**关键优化经验**：
- 积水提示词从"在基坑内"(位置描述)优化为"水面反射光泽"(视觉特征)，F1从44.9%提升至73.47%
- 裸土负例添加"水面反射/湿泥积水"排除项，减少积水误判为裸土

### 3.3 云边协同架构

```
工地摄像头
    │
    ├─ 边缘设备 (Jetson)
    │    └─ YOLOv8-Pose (30 FPS)
    │         ├─ 安全装备检测（4类）
    │         ├─ 危险区域管控（3类）
    │         ├─ 危险行为识别（5类）
    │         └─ 机械与设备（3类）
    │
    └─ 云端 (GPU Server)
         ├─ C-RADIOv4 零样本检测 (5-10 FPS)
         │    ├─ 裸土未覆盖
         │    ├─ 扬尘污染
         │    ├─ 坑内积水
         │    └─ 基坑边材料堆放
         ├─ 数据存储
         ├─ 统计分析
         └─ 报表生成
```

### 3.3 危险区域联动

```
检测到机械作业（塔吊/挖掘机）
    │
    ├─ 识别机械位置和作业范围
    │
    ├─ 动态设置危险区域（半径 5-10米）
    │
    └─ 监控人员进入
         └─ 触发 unsafe_distance 报警
```

## 8. 常见问题

### Q1: 为什么施工安全只需要1个模型？

**原因**：
1. 所有目标都有固定形状（人员、装备、机械）
2. 样本充足（现场采集）
3. 不需要开放词汇分割
4. YOLOv8-Pose 可同时处理检测和行为

### Q2: 如何实现危险区域联动？

**方法**：
1. 检测机械（塔吊、挖掘机）
2. 计算作业范围（半径 5-10米）
3. 动态设置危险区域
4. 监控人员进入并报警

### Q3: 安全装备如何同时检测多个？

**方法**：
1. 检测人员
2. 对每个人检测头部和身体
3. 判断是否有安全帽/反光衣
4. 分类到对应类别
