## 1. 系统概述

### 1.1 应用场景

施工安全监测系统用于建筑工地、施工现场等场景的智能安全管理，实现：
- 人员安全装备检测
- 施工区域管控
- 危险行为识别
- 环境安全监测

### 1.2 核心方案

| 模型 | 任务 | 类别 | 数据集 | FPS | 位置 |
|------|------|------|--------|-----|------|
| **YOLOv8-Pose** | 人员检测+行为 | 15类 | 5500张 | 30+ | 边缘 |

### 1.3 特殊说明

**为什么施工安全只需要1个模型？**

| 对比项 | 水利巡检 | 园区监测 | 施工安全 |
|--------|---------|---------|---------|
| **目标特征** | 部分无固定形状 | 都有固定形状 | 都有固定形状 |
| **样本稀缺** | 水质极稀缺 | 充足 | 充足 |
| **模型需求** | 需要 DINOv3+SAM3 | 2个 YOLO | 1个 YOLO |

**原因**：
- 所有检测目标都有固定形状（人员、装备、机械）
- 样本充足（可通过现场采集获得）
- 不需要开放词汇分割

## 3. 技术方案

### 3.1 YOLOv8-Pose 架构

```
输入图像 (640×640)
    │
    ├─ Backbone (CSPDarknet)
    ├─ Neck (PANet)
    ├─ Head (Detect) → 15类检测
    └─ Pose Head → 关键点（用于行为分析）
```

### 3.2 边缘部署架构

```
工地摄像头
    │
    ├─ 边缘设备 (Jetson)
    │    └─ YOLOv8-Pose (30 FPS)
    │         ├─ 安全装备检测（4类）
    │         ├─ 危险区域管控（3类）
    │         ├─ 危险行为识别（5类）
    │         └─ 机械与设备（3类）
    │
    └─ 云端（可选）
         ├─ 数据存储
         ├─ 统计分析
         └─ 报表生成
```

### 3.3 危险区域联动

```
检测到机械作业（塔吊/挖掘机）
    │
    ├─ 识别机械位置和作业范围
    │
    ├─ 动态设置危险区域（半径 5-10米）
    │
    └─ 监控人员进入
         └─ 触发 unsafe_distance 报警
```

## 8. 常见问题

### Q1: 为什么施工安全只需要1个模型？

**原因**：
1. 所有目标都有固定形状（人员、装备、机械）
2. 样本充足（现场采集）
3. 不需要开放词汇分割
4. YOLOv8-Pose 可同时处理检测和行为

### Q2: 如何实现危险区域联动？

**方法**：
1. 检测机械（塔吊、挖掘机）
2. 计算作业范围（半径 5-10米）
3. 动态设置危险区域
4. 监控人员进入并报警

### Q3: 安全装备如何同时检测多个？

**方法**：
1. 检测人员
2. 对每个人检测头部和身体
3. 判断是否有安全帽/反光衣
4. 分类到对应类别
