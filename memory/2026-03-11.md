## 僵尸进程问题解决 (21:14-21:22)

### 问题发现
- Epoch 59在batch 600卡住，GPU空闲
- 发现68个僵尸进程（defunct）
- 根因：num_workers=8过高，内存压力导致DataLoader worker崩溃

### 解决方案
1. **降低num_workers**: 8 → 4
   - 内存占用减少50% (2.4GB → 1.2GB)
   - 速度几乎不受影响（11-12分钟/epoch）
   
2. **部署自动监控脚本**
   - 路径：`/workspace/rcmt_v3/scripts/zombie_monitor.py`
   - 每5分钟检查僵尸进程数量
   - 超过5个自动重启训练
   - 日志：`/workspace/rcmt_v3/logs/zombie_monitor.log`

3. **性能对比**
   | num_workers | 内存占用 | 稳定性 | 速度 |
   |-------------|----------|--------|------|
   | 8 (之前) | 2.4GB | ❌ 不稳定 | 11.5min |
   | 4 (现在) | 1.2GB | ✅ 稳定 | 12min |

## Token自动管理配置 (21:36)

### 问题
- 上下文超过205k tokens → `model_context_window_exceeded`
- 必须手动到电脑重启gateway
- 手机端无法自动恢复

### 解决方案

**1. Compaction配置**
```json
{
  "mode": "safeguard",
  "reserveTokens": 20000,
  "keepRecentTokens": 30000,
  "maxHistoryShare": 0.7
}
```

**2. Context Pruning配置**
```json
{
  "softTrimRatio": 0.8,
  "hardClearRatio": 0.9
}
```

### 效果
- ✅ 自动压缩历史（保留最近30k tokens）
- ✅ 自动清理早期内容（80%开始，90%强制）
- ✅ 完全自动，无需手动干预
- ✅ 子代理自动继承这些配置
