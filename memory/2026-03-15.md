## 03:12 - RCMT Training Container Restart

### Issue: Zombie Processes
- Found 4 zombie workers (PID 16, 17, 18, 19) that completed training but never exited
- Parent process (PID 1) was `sleep infinity` - doesn't reap zombies
- Zombies were consuming no resources but cluttering process list

### Solution
- Restarted container: `docker restart rcmt-training`
- Training auto-restarts via container initialization script
- Entry point runs: `cd /workspace/rcmt_v3 && nohup python3 train_rcmt_v6_paper.py ...`

### Post-Restart Status (03:04)
- All processes healthy (main + 4 workers)
- GPU utilization: 97%
- Memory: 15,929 / 16,311 MB
- No zombie processes

### Key Learning
- Container uses `sleep infinity` as PID 1, which doesn't reap zombie processes
- Long-running training can accumulate zombies from completed worker processes
- Periodic container restart helps clean up
