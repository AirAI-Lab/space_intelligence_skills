### 4. Patch分类调参

**问题诊断**:
- best_patch_overlap 普遍偏低 (平均 < 0.35)
- 分割质量较差 (seg_ok: 34.3%)

**优化策略**:
- ✅ 提高min_overlap_threshold: 0.30 → 0.50
- ✅ 增加patch采样密度: 默认 → 64 patches
- ✅ 优化patch选择策略: top-k策略，选择置信度最高的10个patches
- ✅ 增加overlap权重: 1.5x

**修改内容**:
```yaml
patch:
  num_patches: 64                    # 提高采样密度
  min_overlap_threshold: 0.50        # ⭐ 关键：从0.30提高到0.50
  patch_selection_strategy: "top_k"  # 选择top-k patches
  top_k_patches: 10                  # 选择置信度最高的10个
  overlap_weight: 1.5                # 增加overlap权重
```

**预期效果**:
- Patch覆盖率提升: 平均overlap从<0.35 → >0.50
- 分割质量改善: seg_ok从34.3% → 50%+
- 减少误检: 通过top-k策略过滤低置信度patches
