# OpenClaw 飞书文件生成使用指南

## 问题说明
飞书/WhatsApp 等聊天通道中，AI **无法直接创建文件**（工具限制）。

## 解决方案

### 方案 1：使用 WebChat 界面（推荐）✅

WebChat 支持完整的工具集（read, write, edit）：

```bash
# 启动 Gateway
cd d:/github/edge_infer_cloud
openclaw gateway --port 18789 --token edge-infer-claw-2026

# 访问 http://localhost:18789
# 在 WebChat 中直接让 AI 创建文件
```

**示例对话：**
```
你：帮我创建一个文件 d:/github/edge_infer/models/rcmt/net/test.py
AI：[使用 write_file 工具创建文件] ✅
```

### 方案 2：飞书 + 手动复制

在飞书中让 AI 生成代码块，然后手动复制：

**飞书对话：**
```
你：生成 RegionTemporalConsistency 类的完整代码
AI：```python
class RegionTemporalConsistency(nn.Module):
    ...
```
```

然后复制代码到本地文件。

### 方案 3：使用 /exec 命令（高级）

在飞书中使用特殊命令格式：

```
/exec write d:/github/edge_infer/models/rcmt/net/test.py
```python
def hello():
    print("Hello")
```
```

## 配置飞书快捷命令

创建自定义快捷命令：

1. 编辑 `~/.openclaw/openclaw.json`
2. 添加自定义命令映射
3. 重启 Gateway

## 文件路径规则

AI 只能在以下路径创建文件：
- `d:/github/edge_infer/`
- `d:/github/edge_infer_cloud/`

## 常用命令

| 命令 | 说明 |
|------|------|
| `/status` | 查看状态 |
| `/new` | 重置会话 |
| `/help` | 查看帮助 |

## 推荐工作流

1. **开发阶段**：使用 WebChat (http://localhost:18789) - 支持所有工具
2. **移动端**：使用飞书 - 获取代码后手动复制
3. **快速查询**：使用飞书/WhatsApp - 简单问答
