# 智能委托系统 (Auto-Delegate)

## 核心功能

**全自动智能判断**：系统自动分析每个任务的复杂度，决定是在主对话执行还是委托给子代理。

用户无需学习任何特殊语法，**自然输入即可**，系统自动优化。

---

## 智能判断标准

系统自动检测以下指标：

### 指标1：文件操作复杂度
```
简单：1-2个文件 -> 主对话执行
中等：3-5个文件 -> 考虑子代理
复杂：6+个文件  -> 自动子代理
```

### 指标2：工具调用预估
```
简单：1-3次工具调用 -> 主对话执行
中等：4-8次工具调用 -> 考虑子代理
复杂：9+次工具调用  -> 自动子代理
```

### 指标3：任务类型检测
```
自动子代理的场景：
- 论文修订/写作
- 代码审查/重构
- 批量文件处理
- 多步骤分析
- 长时间研究
```

### 指标4：关键词检测
```
高权重关键词（每个+3分）：
- "全面"、"完整"、"所有"、"深度"、"详细"
- "修订"、"优化"、"重构"、"分析"、"研究"
- "批量"、"多个"、"所有文件"

中权重关键词（每个+2分）：
- "配置"、"集成"、"结合"
- "生成"、"输出"、"报告"

阈值：>= 5分 -> 自动子代理
```

## 自动执行的流程

### 步骤1：任务分析（透明）
系统在后台分析：
- 任务类型
- 涉及文件数
- 预估时间
- 复杂度分数

### 步骤2：决策（透明）
```
if (复杂度分数 >= 5) {
    -> 使用子代理
} else {
    -> 主对话执行
}
```

### 步骤3：执行（自动）
- **主对话**：直接执行，立即返回结果
- **子代理**：后台执行，完成后公告

### 步骤4：结果汇总
- 子代理完成后，自动将结果摘要公告回主对话
- 用户获得清洁的最终结果

## 配置

在 openclaw.json 中配置：

```json
{
  "agents": {
    "defaults": {
      "subagents": {
        "model": "zai/glm-4.7-flash",
        "maxConcurrent": 4,
        "runTimeoutSeconds": 900,
        "autoDelegateThreshold": 5
      }
    }
  }
}
```

### 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| autoDelegateThreshold | 自动委托阈值 | 5 |
| model | 子代理使用的模型 | zai/glm-4.7-flash |
| runTimeoutSeconds | 默认超时时间 | 900 |

## 技术实现

系统对每个任务进行以下分析：

```javascript
function analyzeTask(task) {
  let score = 0;

  // 关键词检测
  score += countKeywords(task, HIGH_PRIORITY) * 3;
  score += countKeywords(task, MEDIUM_PRIORITY) * 2;

  // 文件操作检测
  score += countFileOperations(task) * 2;

  // 工具调用预估
  score += estimateToolCalls(task);

  // 任务类型检测
  if (isComplexTaskType(task)) score += 3;

  return {
    score,
    shouldDelegate: score >= threshold,
    estimatedTime: score * 2 // 分钟
  };
}
```
