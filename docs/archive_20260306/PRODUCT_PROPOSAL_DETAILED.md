## 1. 产品经理智能体 (Product Manager Agent)

### 1.1 智能体职责

**核心职责**：
1. **产品规划** (Product Planning): 制定产品路线图、功能优先级、里程碑
2. **需求管理** (Requirement Management): 收集、分析、管理用户需求
3. **项目协调** (Project Coordination): 协调各智能体、确保项目进度
4. **市场分析** (Market Analysis): 分析市场趋势、竞品、机会
5. **商业策略** (Business Strategy): 制定商业模式、定价策略、市场策略

**输出内容**：
- 产品路线图 (3年、5年、10年)
- 功能需求文档 (PRD)
- 市场分析报告
- 商业策略文档
- 项目里程碑计划

### 10.2 智能体通讯协议

#### 10.2.1 消息类型

**同步消息**（Direct Message）：
- 用途：简单请求-响应
- 延迟：<10ms
- 路由：直接路由，不经过LLM

**异步消息**（Async Message）：
- 用途：复杂处理任务
- 延迟：<100ms
- 路由：消息队列

**复杂消息**（Complex Message）：
- 用途：需要LLM理解和决策
- 延迟：<1s
- 路由：经过LLM

#### 10.2.2 消息格式

**同步消息格式**：
```json
{
  "message_id": "uuid-v4",
  "from_agent": "ui_ux_design_agent",
  "to_agent": "product_manager_agent",
  "message_type": "direct",
  "timestamp": "2026-02-16T10:00:00Z",
  "payload": {
    "type": "ui_design_request",
    "data": {
      "screen_type": "spatial_map",
      "interaction_type": "3d_manipulation",
      "requirements": "..."
    }
  }
}
```

**复杂消息格式**：

### 10.3 智能体决策流程

**核心理念**：智能体不直接依赖LLM进行决策，而是基于本地模型进行快速决策，只有遇到无法处理的情况才调用LLM。

**决策流程**：
1. 本地快速决策（基于小模型、规则、算法）
2. 本地中等决策（基于历史经验、知识库）
3. LLM辅助决策（基于大语言模型）

### 10.4 智能体合并输入输出

**核心理念**：多个智能体的输入和输出通过智能体总线进行合并，支持协同决策和分布式处理。

**合并流程**：
1. 输入收集（Input Collection）
2. 输入预处理（Input Preprocessing）
3. 智能体决策（Agent Decision）
4. 输出后处理（Output Postprocessing）
5. 输出分发（Output Distribution）
