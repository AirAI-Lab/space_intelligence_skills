- ✅ 预算充足（应急管理专项资金）

- ✅ 市场成熟（政策支持、采购渠道成熟）
- ✅ 技术匹配（实时性要求高，适合边缘推理）

- ✅ 市场最大（建筑工地数量多）

- ⚠️ 单价较低（需要规模化）

**Phase 3 - 城管+国土**（评分4.3/4.0）
- ✅ 预算稳定（城市管理专项）
- ✅ 需求明确（违建查处、市容管理）
- ⚠️ 算法复杂（变化检测、语义分割）

**Phase 4 - 环保+电力**（评分3.8/3.7）
- ✅ 政策驱动（环保政策趋严）

**推理逻辑** (`detector.py`):

```python
class EmergencyDetector:
    def __init__(self, config):
        # 加载边缘模型
        self.disaster_model = load_trt_engine(config['edge_models'][0])
        self.rescue_model = load_trt_engine(config['edge_models'][1])
        
        # 云端客户端
        self.cloud_client = CloudClient(config['cloud_services'])
        
    def detect(self, image, task_type='full'):
        """
        应急检测主入口
        
        Args:
            image: 输入图像
            task_type: 'disaster', 'rescue', 'full'
        
        Returns:
            {
                'disaster': [...],  # 灾害检测结果
                'rescue': [...],    # 救援检测结果
                'cloud': {...}      # 云端分析结果
            }
        """
        results = {}
        
        # 边缘实时推理
        if task_type in ['disaster', 'full']:
            results['disaster'] = self.disaster_model.infer(image)
            
        if task_type in ['rescue', 'full']:
            results['rescue'] = self.rescue_model.infer(image)
            
        # 云端深度分析（异步）
        if self.need_cloud_analysis(results):
            results['cloud'] = await self.cloud_client.analyze(
                image, 
                results
            )
            
        return results
    
    def need_cloud_analysis(self, edge_results):
        """判断是否需要云端分析"""
        # 如果边缘检测到灾害，触发云端受灾评估
        if edge_results.get('disaster'):
            return True
        return False
```

#### 3.1.7 云边协同API

**边缘上报API**:

```python
# edge_infer_cloud/api/routes/emergency.py

@router.post("/emergency/report")
async def emergency_report(
    device_id: str,
    image: UploadFile,
    edge_results: dict,  # 边缘推理结果
    location: dict       # GPS位置
):
    """
    应急事件上报
    - 边缘检测到灾害/救援目标后上报
    - 云端进行深度分析
    """
    # 1. 保存图像
    image_url = await save_image(image, device_id)
    
    # 2. 云端深度分析
    cloud_result = await cloud_analysis(
        image_url,
        edge_results,
        location
    )
    
    # 3. 告警通知
    if cloud_result['severity'] == 'high':
        await send_alert(device_id, cloud_result)
    
    # 4. 保存到数据库
    await save_emergency_event(device_id, cloud_result)
    
    return {"status": "success", "result": cloud_result}

@router.get("/emergency/assess")
async def disaster_assess(
    device_id: str,
    assessment_type: str  # 'area', 'earthquake', 'typhoon'
):
    """
    灾害评估API
    - 受灾面积评估
    - 地震损毁评估
    - 台风灾害评估
    """
    # 调用RCMT变化检测
    result = await rcmt_assess(device_id, assessment_type)
    
    return {"assessment": result}
```

## 8. 总结

### 8.1 核心差异

| 维度 | 技术导向（错误） | 业务导向（正确） |
|------|----------------|----------------|
| **分组依据** | 技术特点（边缘/云端/YOLO） | 业务部门（应急/住建/国土...） |
| **算法包** | 通用包（实时组/周期组） | 部门定制包（12种/15种/10种...） |
| **实施顺序** | 技术难易度 | 市场价值评分 |
| **客户理解** | 技术术语（模型、推理时间） | 业务术语（灾害预警、施工安全） |
| **销售难度** | 高（需技术解释） | 低（直接对话业务部门） |

### 8.2 关键成功因素

1. ✅ **产品工程师主导需求**：深入了解每个部门的真实业务场景
2. ✅ **算法工程师按部门定制**：不是通用模型，而是部门专属模型
3. ✅ **软件工程师快速集成**：每个部门独立插件，快速部署
4. ✅ **渐进式实施**：Phase 1验证成功后再做Phase 2
5. ✅ **业务价值优先**：先做预算充足、需求刚性的部门
