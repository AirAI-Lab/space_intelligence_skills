#### 算法1.1：山体滑坡检测

**业务场景** → **算法需求** → **模型选择** → **数据集需求**

```
业务场景：
├─ 输入：无人机航拍的山区图像（双时相对比）
├─ 输出：滑坡区域标注 + 风险等级评估
├─ 频率：实时（<1分钟）
└─ 精度：F1 > 0.85

算法需求：
├─ 任务类型：语义分割（变化检测）
├─ 检测目标：山体滑坡区域
├─ 时序要求：双时相对比
└─ 实时性：边缘推理 <60s

模型选择：
├─ 方案1：RCMT V3-Swin（变化检测专用）
│  ├─ 优势：时序建模强，精度高
│  ├─ 劣势：模型大，推理慢
│  └─ 适用：云端推理（<2s）
├─ 方案2：YOLOv8-Seg（实时分割）
│  ├─ 优势：边缘部署快，实时性好
│  ├─ 劣势：精度略低
│  └─ 适用：边缘推理（<100ms）✅
└─ 最终选择：YOLOv8-Seg（边缘）+ RCMT（云端深度分析）

数据集需求：
├─ 任务：滑坡区域分割
├─ 训练集：2,000张
├─ 验证集：300张
├─ 测试集：300张
├─ 标注类型：像素级分割掩码
├─ 数据来源：
│  ├─ 公开数据集：LEVIR-CD（建筑变化检测）⚠️ 不完全匹配
│  ├─ 自建数据：历史滑坡航拍数据 ✅
│  └─ 合成数据：滑坡仿真（3D建模）✅
└─ 标注成本：¥3,000（中等难度）
```

**数据获取方案**:

**方案A：公开数据集（30%）**

| 数据集 | 网址 | 数据量 | 匹配度 | 说明 |
|--------|------|--------|--------|------|
| LEVIR-CD | GitHub | 637对 | ⭐⭐ | 建筑变化检测，可迁移 |
| SYSU-CD | 学术论文 | 20,000对 | ⭐⭐⭐ | 建筑变化检测 |
| WHU-CD | GitHub | 1,608对 | ⭐⭐ | 建筑变化检测 |
| CDD | GitHub | 10,000对 | ⭐⭐⭐ | 多类别变化检测 |

**使用策略**:
- 用于预训练模型（迁移学习）
- 提供变化检测的基础能力
- 需要在滑坡数据上微调

**方案B：自建数据集（50%）**

**数据来源**:
1. **政府公开数据**
   - 自然资源部：地质灾害历史数据
   - 应急管理部：灾害评估报告
   - 地质调查局：滑坡案例库

2. **新闻媒体数据**
   - 灾害新闻报道配图
   - 航拍视频截图
   - 网络爬虫（需注意版权）

3. **无人机采集**
   - 合作无人机公司
   - 灾害现场实地采集
   - 历史灾后影像

**采集计划**:
```bash
第1周：联系应急管理厅，申请历史数据
第2周：网络爬虫收集新闻图片（500张）
第3周：数据清洗与筛选（保留300张）
第4周：补充无人机采集（200张）
总计：1,000张高质量数据
```

**方案C：合成数据（20%）**

**工具**: Blender + Google Earth Studio

**流程**:
```
1. 3D地形建模（DEM数据）
2. 滑坡仿真（物理引擎）
3. 航拍视角渲染
4. 双时相合成
```

**产出**: 400张合成数据

**优势**:
- 完全可控（滑坡形态、光照、视角）
- 零成本
- 可扩展（无限生成）

**劣势**:
- 与真实数据有domain gap
- 需要域适应训练

**数据标注方案**:

**标注工具**: Label Studio / CVAT

**标注类型**: 像素级分割掩码

**标注规范**:
```
类别定义：
- landslide: 山体滑坡区域（土壤裸露、地形变化）
- non_landslide: 非滑坡区域

标注要求：
- 精确到像素级别
- 边界清晰（避免模糊区域）
- 双时相标注（T1和T2）
```

**标注流程**:
```
1. 标注员初标注（2天，500张）
2. 审核员审核（1天，纠错率<10%）
3. 专家验收（1天，重点样本）
4. 质量抽查（0.5天，10%样本）
总计：4.5天
```

**标注成本**: ¥3,000
- 标注员：¥50/天 × 5人 × 2天 = ¥500
- 审核员：¥100/天 × 2人 × 1天 = ¥200
- 专家：¥300/天 × 1人 × 1天 = ¥300
- 外包标注：2,000张 × ¥1/张 = ¥2,000

```python
import requests
from bs4 import BeautifulSoup
import os

def crawl_news_images(keyword, num_images=500):
    """
    爬取新闻网站灾害图片
    """
    # 百度图片搜索
    url = f"https://image.baidu.com/search/index?tn=baiduimage&word={keyword}"
    headers = {'User-Agent': 'Mozilla/5.0'}
    
    response = requests.get(url, headers=headers)
    soup = BeautifulSoup(response.text, 'html.parser')
    
    img_urls = []
    for img in soup.find_all('img'):
        src = img.get('src')
        if src and 'http' in src:
            img_urls.append(src)
    
    # 下载图片
    for i, img_url in enumerate(img_urls[:num_images]):
        img_data = requests.get(img_url).content
        with open(f'data/{keyword}_{i}.jpg', 'wb') as f:
            f.write(img_data)
    
    return len(img_urls)

# 使用示例
crawl_news_images('山体滑坡', 500)
crawl_news_images('泥石流', 300)
crawl_news_images('洪涝灾害', 300)
```

### 7.2 关键建议

**1. 数据策略调整**

**3. 并行加速**

```diff
- 串行：采集 → 标注 → 训练
+ 并行：采集 || 标注 || 数据准备
+ 时间节省：2个月 → 1.5个月
```

**4. 质量优先**

```diff
- 追求数据量（10,000张）
+ 追求数据质量（5,000张高质量）
+ 标注质量 > 数据量
```
