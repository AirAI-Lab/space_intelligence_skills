## 一、平台整体技术架构

### 1.1 云边协同双引擎架构

本平台采用**边缘实时推理 + 云端深度分析**的双引擎架构，边缘侧负责毫秒级实时检测，云端负责零样本分割与大模型推理，两级协同形成完整的智能视觉分析闭环。

```
┌──────────────────────────────────────────────────────────────┐
│                      云端平台 (edge_infer_cloud)              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐   │
│  │ Spring   │ │ Vue3     │ │ C-RADIOv4│ │ YOLOv8        │   │
│  │ Boot 3.x │ │ 前端     │ │ 零样本   │ │ 训练服务      │   │
│  │ REST API  │ │ 仪表盘   │ │ 分割     │ │ MLflow 管理   │   │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └──────┬────────┘   │
│       │            │            │               │            │
│  ┌────┴────────────┴────────────┴───────────────┴────────┐   │
│  │ TimescaleDB │ Redis │ EMQX MQTT │ SeaweedFS S3        │   │
│  └────────────────────────┬───────────────────────────────┘   │
└───────────────────────────┼───────────────────────────────────┘
                            │ MQTT + REST API
              ┌─────────────┼─────────────┐
              │             │             │
    ┌─────────┴───┐ ┌──────┴──────┐ ┌────┴──────┐
    │ 边缘设备 1   │ │ 边缘设备 2   │ │ 边缘设备 N │
    │ (Jetson)    │ │ (Jetson)    │ │ (Jetson)  │
    │ TensorRT    │ │ TensorRT    │ │ TensorRT  │
    │ Plugin Chain│ │ Plugin Chain│ │ Plugin    │
    └─────────────┘ └─────────────┘ └───────────┘
```

## 二、核心技术优势（控标亮点）

### 2.1 零样本分割能力（独家技术壁垒）

**技术描述**：云端集成 C-RADIOv4-H 大视觉模型（14亿参数），融合三大教师模型：

| 教师模型 | 参数量 | 能力 |
|---------|--------|------|
| SigLIP2-giant | — | 多语言文本-图像对齐 |
| DINOv3 | 7B（70亿） | 自监督视觉特征提取 |
| SAM3 | — | 通用分割能力 |

**核心优势**：
- **无需训练数据**：仅通过自然语言文本描述即可定义新检测类别
- **即开即用**：从文本描述到上线检测，无需数据采集、标注、训练周期
- **中文支持**：支持中文类别名称直接定义（如"裸土未覆盖"、"积尘"）
- **混合推理**：SigLIP2 语义分类 + RADIO 异常分割双通道，精度与召回兼顾

**控标角度**：
> 投标要求中可写入"支持零样本视觉分割，无需采集训练数据即可新增检测场景，新增场景上线时间≤1小时"。传统方案需求数据采集(1-2周) + 标注(1-2周) + 训练(1-3天) + 部署(1天)，本平台将新增场景周期从**4-5周缩短至1小时内**。

### 2.2 动态插件链架构（边缘侧核心壁垒）

**技术描述**：边缘推理引擎采用动态加载插件链架构，16个专用AI插件作为独立共享库（.so），支持运行时按需组合。

**插件清单**（16个）：

| 类别 | 插件名称 | 功能 |
|------|---------|------|
| 安全检测 | helmet_detect | 安全帽佩戴检测（人-帽关联） |
| 安全检测 | person_detection | 人员检测 |
| 安全检测 | vehicle_detection | 车辆检测（轿车/卡车/客车等） |
| 安全检测 | face_detection | 人脸检测 |
| 安全检测 | thermal_detection | 热成像分析 |
| 通用检测 | object_detection | 通用目标检测（80类） |
| 分析跟踪 | mot_tracking | 多目标跟踪（MOT） |
| 分析跟踪 | counting | 目标计数与密度分析 |
| 分析跟踪 | change_detection | 场景变化检测 |
| 分析跟踪 | anomaly_detection | 异常行为检测 |
| 高级分析 | instance_seg | 实例分割 |
| 高级分析 | semantic_seg | 语义分割 |
| 高级分析 | pose_estimation | 人体姿态估计 |
| 识别类 | text_ocr | 文字识别 |
| 识别类 | lp_recognition | 车牌识别 |
| 后处理 | postprocess_yolov8 | YOLOv8 专用后处理 |

**核心优势**：
- **热加载**：运行时动态加载/卸载插件，无需重启服务
- **链式组合**：多个插件可按需串联，形成复杂分析流水线
- **标准接口**：统一的 C++ Plugin API，新增插件只需实现3个接口方法
- **独立编译**：每个插件独立编译为 .so，互不影响

**控标角度**：
> 投标要求中可写入"边缘推理引擎支持≥16种AI分析插件，支持运行时热加载，插件可通过标准API接口自主开发扩展"。传统方案通常是硬编码检测逻辑，新增功能需重新编译整个系统。

### 2.3 OTA 空中升级（运维效率壁垒）

**技术描述**：基于 MQTT 的完整 OTA 升级流水线，支持远程模型部署、版本管理、进度追踪。

**OTA 流程**：
```
云端管理平台 → MQTT命令下发 → 边缘接收 → HTTP下载模型 → MD5校验
→ ONNX转TensorRT → 热加载生效 → 状态回传
```

**核心优势**：
- **零停机升级**：模型热替换，推理服务不中断
- **完整性校验**：MD5 校验确保传输可靠
- **自动转换**：ONNX 自动转换为 TensorRT 引擎，适配当前硬件
- **回滚支持**：升级失败自动回退至上一版本
- **批量升级**：支持同时向多台边缘设备推送升级
- **实时进度**：WebSocket 推送每台设备的升级状态

**控标角度**：
> 投标要求中可写入"支持OTA空中升级，模型更新无需现场维护，支持批量推送、自动校验、热加载生效、失败自动回滚"。传统方案需人工到现场逐台更新。

### 2.4 云边双级告警体系

**技术描述**：边缘侧实时告警 + 云端深度分析告警的双级告警体系，支持自定义规则和外部推送。

**告警链路**：
```
边缘侧实时检测 → 本地插件链判警 → MQTT上报 → 云端存储/推送
                    ↓ (可选)
              触发云端深度分析 → C-RADIOv4 零样本分割 → 二次告警
```

**核心优势**：
- **三级告警**：critical / warning / info 分级管理
- **自定义规则**：按面积阈值、置信度阈值、目标数量灵活配置
- **Webhook 推送**：告警自动推送至第三方平台，支持 HMAC 签名验证
- **实时推送**：WebSocket 广播，前端秒级刷新
- **关联触发**：边缘告警可自动触发云端二次分析

**控标角度**：
> 投标要求中可写入"支持边缘-云端双级告警体系，边缘侧毫秒级实时告警可自动触发云端二次分析，告警规则可按设备/模型/类别/阈值灵活配置，支持 Webhook 对接第三方系统"。

### 2.5 全链路模型管理（MLOps 壁垒）

**技术描述**：从数据标注到模型部署的全链路模型生命周期管理。

**MLOps 流水线**：
```
数据集管理 → YOLOv8 训练 → MLflow 实验追踪 → 模型版本管理
→ ONNX 导出 → TensorRT 转换 → OTA 部署 → 推理监控
```

**核心优势**：
- **训练服务内置**：平台自带 YOLOv8 训练服务，支持自定义数据集训练
- **实验追踪**：MLflow 自动记录 mAP50、mAP50-95、Precision、Recall、Loss 曲线
- **模型版本管理**：多版本模型共存，支持 A/B 测试和回滚
- **自动转换**：PyTorch → ONNX → TensorRT 自动化转换管线
- **智能续训**：断点续训、Early Stopping、超参调优

### 5.2 零样本扩展能力

通过文本描述即可新增检测类别，**无需任何训练数据**：

```json
{
  "class_name": "fire_smoke",
  "class_name_cn": "烟火",
  "text_prompt": "smoke or fire in the scene"
}
```

新增一个检测类别的完整流程仅需：
1. 配置文本描述（1 分钟）
2. 平台自动加载（即时生效）
3. 告警规则配置（可选，5 分钟）

## 六、控标建议条款

### 6.1 技术规格控标点

以下条款建议写入技术评分项或实质性要求：

1. **边缘推理引擎**
   - ✓ 支持 TensorRT 加速推理，推理延迟 ≤35ms（YOLOv8n, Jetson Orin）
   - ✓ 支持 FP16/INT8 量化推理
   - ✓ 支持 ≥16 种 AI 分析插件的动态加载和链式组合
   - ✓ 支持 ONNX → TensorRT 自动转换和热加载
   - ✓ 支持 RTMP/RTSP/本地文件多种视频源接入

2. **云端零样本分析**
   - ✓ 支持零样本语义分割，无需训练数据即可新增检测场景
   - ✓ 新增检测类别上线时间 ≤1 小时
   - ✓ 支持中文自然语言定义检测类别
   - ✓ 集成多教师模型融合（视觉特征 + 文本对齐 + 分割）

3. **云边协同**
   - ✓ 支持 MQTT 协议设备管理与消息通信
   - ✓ 支持 OTA 空中升级，模型热替换无需停机
   - ✓ 支持边缘告警触发云端二次深度分析
   - ✓ 支持 WebSocket 实时数据推送
   - ✓ 支持时序数据库存储推理结果，自动分区压缩

4. **模型管理**
   - ✓ 内置 YOLOv8 训练服务，支持自定义数据集训练
   - ✓ 集成 MLflow 实验追踪，自动记录 mAP/Precision/Recall/Loss
   - ✓ 支持模型版本管理和 A/B 测试
   - ✓ 支持 PyTorch → ONNX → TensorRT 全链路转换

5. **告警与集成**
   - ✓ 支持三级告警（critical/warning/info）和自定义规则
   - ✓ 支持 Webhook 对接第三方系统，含 HMAC 签名验证
   - ✓ 支持 S3 兼容对象存储接口
   - ✓ 提供完整 REST API，含 Swagger/OpenAPI 文档

### 6.2 加分项建议

- 全部核心组件采用 Apache 2.0/MIT 协议，无 GPL 传染风险
