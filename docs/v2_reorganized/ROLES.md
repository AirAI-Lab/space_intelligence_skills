> **更新时间**：2026-03-06

## 六大核心角色

### 🧮 算法工程师

**职责**：AI算法研发、模型训练、感知/认知/行动算法

**快捷指令**：
```bash
@delegate[算法工程师] <任务描述>
```

**典型任务**：
```bash
@delegate[算法工程师] 优化RCMT模型训练参数
@delegate[算法工程师] 设计路径规划算法
@delegate[算法工程师] 开发目标检测模型
@delegate[算法工程师] 实现多机协同决策算法
@delegate[算法工程师] 优化模型推理延迟
```

**覆盖范围**：
- 感知算法（目标检测、变化检测、语义分割）
- 认知算法（路径规划、任务分配、风险评估）
- 行动算法（飞行控制、避障导航）
- 模型优化（量化压缩、推理加速）

**关键指标**：
- RCMT F1分数 > 0.92
- 推理延迟 ≤ 30ms
- 模型精度和性能

## 使用规则

### ✅ 使用角色子代理的场景

1. **任务明确归属某个角色**
   ```
   @delegate[硬件工程师] 评估妙算3的功耗方案
   ```

2. **需要专业知识分析**
   ```
   @delegate[算法工程师] 优化模型训练参数
   ```

3. **需要快速获取专业建议**
   ```
   @delegate[软件工程师] 分析系统性能瓶颈
   ```

---

### ⚡ 使用通用子代理的场景

1. **任务未指定角色**
   ```
   @delegate 分析这个PR的代码变更
   ```

2. **临时性、一次性任务**
   ```
   @delegate 生成今天的工作日报
   ```

3. **快速查询或对比**
   ```
   @delegate 对比两个方案的成本
   ```

## 角色职责边界

### 算法工程师 vs 软件工程师

```
算法工程师：做什么（What）
- 模型设计和训练
- 算法研发和优化
- 性能指标提升

软件工程师：怎么做（How）
- 系统架构设计
- 代码实现和测试
- 工程化落地
```

### 硬件工程师 vs 软件工程师

```
硬件工程师：物理层
- PCB设计
- 硬件集成
- 功耗和散热

软件工程师：逻辑层
- 驱动开发
- 接口封装
- 性能优化
```

### 产品工程师 vs 负责人

```
产品工程师：产品和用户
- 功能规划
- 用户体验
- 文档维护

负责人：项目和团队
- 项目统筹
- 资源协调
- 风险管理
```

## 注意事项

1. **角色专一性**：每个角色专注于自己的领域
2. **任务明确性**：调用时尽量明确任务目标
3. **结果验证**：主session应该验证子代理返回的结果
4. **历史追溯**：所有对话都保存在主session历史中
