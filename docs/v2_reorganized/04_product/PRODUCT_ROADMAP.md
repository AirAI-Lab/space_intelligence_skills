## 1. 产品定位

### 1.1 一句话定位

> **让无人机具备空间感知、认知和行动能力**

### 1.2 核心理念

**从"工具"到"伙伴"**

| 传统无人机 | 空中空间智能体 |
|-----------|---------------|
| 远程控制的工具 | 自主思考的伙伴 |
| 执行预定任务 | 理解环境并决策 |
| 单机作业 | 多机协同 |
| 静态能力 | 持续学习进化 |

### 1.3 核心价值主张

```
用户场景 → 需要的能力 → 我们提供的能力 → 用户价值
   ↓           ↓              ↓              ↓
"区域有什么变化？" → 变化检测能力 → 感知智能体 → 快速发现异常
"怎么巡检最高效？" → 路径规划能力 → 认知智能体 → 节省时间成本
"多机如何配合？" → 协同作业能力 → 行动智能体 → 提升作业效率
```

## 2. 核心智能体架构

### 2.1 三大核心智能体

我们采用**3+1**的智能体架构：3个核心智能体 + 1个云边协同平台。

```
┌─────────────────────────────────────────────────────────────┐
│                    空中空间智能体平台                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   云边协同平台                        │   │
│  │     设备管理 | 模型管理 | 训练服务 | OTA升级          │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↕                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │              │  │              │  │              │     │
│  │  感知智能体   │  │  认知智能体   │  │  行动智能体   │     │
│  │              │  │              │  │              │     │
│  │  Perception  │  │  Cognition   │  │   Action     │     │
│  │              │  │              │  │              │     │
│  │    "看"      │  │    "想"      │  │    "做"      │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 智能体职责定义

| 智能体 | 核心职责 | 用户视角 | 能力模块 |
|--------|----------|----------|----------|
| **感知智能体** | 理解环境 | "无人机能看到什么" | 目标识别、变化检测、语义分割、三维重建 |
| **认知智能体** | 分析决策 | "无人机理解什么" | 空间推理、路径规划、任务分配、风险评估 |
| **行动智能体** | 执行任务 | "无人机能做什么" | 自主飞行、协同作业、人机交互 |

### 2.3 为什么是3个而不是9个？

**之前的设计问题**：
```
❌ 空间感知Agent + 空间推理Agent + 空间交互Agent + 
   学习Agent + 任务规划Agent + 协作Agent + 
   通讯Agent + 用户交互Agent + 监控Agent = 9个碎片化Agent
```

**问题分析**：
- 职责重叠（任务规划 vs 空间推理）
- 用户无法理解
- 实现复杂度过高
- 维护成本过高

**优化后的设计**：
```
✅ 感知智能体 + 认知智能体 + 行动智能体 = 3个核心智能体
```

**优势**：
- 清晰的"看-想-做"逻辑
- 用户易于理解
- 模块化内部能力
- 渐进式开发

## 3. 能力矩阵

### 3.1 感知智能体能力

**核心使命**：让无人机"看见"并"理解"环境

| 能力 | 描述 | 应用场景 | 技术实现（可替换） |
|------|------|----------|-------------------|
| **目标识别** | 识别人、车、建筑等目标 | 安防监控、搜救 | YOLOv8, RT-DETR |
| **变化检测** | 检测环境时序变化 | 违建监测、灾害评估 | RCMT, ChangeFormer |
| **语义分割** | 像素级场景理解 | 地物分类、植被分析 | SegFormer, UPerNet |
| **三维重建** | 构建空间三维模型 | 地形测绘、资产数字化 | COLMAP, NeRF |
| **目标跟踪** | 跨帧目标关联 | 行为分析、轨迹追踪 | DeepSORT, ByteTrack |
| **关键点检测** | 人体姿态估计 | 安全合规检测 | YOLOv8-Pose |

**重要说明**：技术实现列出的算法只是当前选择，用户无需关心具体算法，只需使用能力。

### 3.2 认知智能体能力

**核心使命**：让无人机"思考"并"规划"

| 能力 | 描述 | 应用场景 | 技术实现 |
|------|------|----------|----------|
| **空间推理** | 基于感知结果进行推理 | 态势分析 | 规则引擎 + LLM |
| **路径规划** | 最优飞行路径计算 | 巡检任务 | A*, RRT* |
| **任务分配** | 多机任务协调 | 大规模作业 | 拍卖算法 |
| **风险评估** | 飞行安全评估 | 自主决策 | 决策树 |
| **变化分析** | 变化趋势和原因分析 | 长期监测 | 时序分析 |

### 3.3 行动智能体能力

**核心使命**：让无人机"执行"并"协同"

| 能力 | 描述 | 应用场景 | 技术实现 |
|------|------|----------|----------|
| **自主飞行** | 执行飞行任务 | 自动巡检 | 飞控SDK |
| **协同作业** | 多机协同执行 | 大范围搜索 | Mesh通信 |
| **避障导航** | 动态障碍物避让 | 复杂环境 | 视觉避障 |
| **人机交互** | 与操作员协作 | 远程操控 | 自然语言/手势 |
| **应急处理** | 异常情况响应 | 安全保障 | 预案库 |

## 4. 应用场景

### 4.1 场景驱动的产品思维

**我们不是从技术出发，而是从用户场景出发**

```
用户说："我需要监控这片区域有没有违建"
    ↓
分析需求：需要"变化检测"能力
    ↓
产品方案：感知智能体 + 认知智能体（变化分析）
    ↓
技术选型：RCMT或ChangeFormer（用户无需关心）
```

### 4.2 典型场景映射

| 场景 | 用户需求 | 需要的能力 | 智能体组合 |
|------|----------|------------|-----------|
| **电力巡检** | 发现线路缺陷和周边隐患 | 目标识别 + 变化检测 | 感知 |
| **城市违建监测** | 检测新增建筑物 | 变化检测 + 语义分割 | 感知 + 认知 |
| **农业监测** | 作物长势和病虫害 | 变化检测 + 语义分割 | 感知 + 认知 |
| **应急救援** | 快速定位受困人员 | 目标识别 + 路径规划 | 感知 + 认知 + 行动 |
| **边境巡逻** | 持续监控异常活动 | 目标识别 + 变化检测 + 协同 | 感知 + 认知 + 行动 |
| **工地安全** | 检测安全帽、反光衣 | 目标识别 + 关键点检测 | 感知 |
| **测绘成图** | 生成三维地图 | 三维重建 | 感知 |

> **技术是为能力服务的，用户不需要关心技术细节**

| 原则 | 说明 |
|------|------|
| **能力优先** | 用户选择能力，系统选择技术 |
| **可替换性** | 技术实现可以随时升级替换 |
| **透明化** | 用户无需了解底层算法 |

## 10. 总结

### 10.1 核心设计理念

1. **场景驱动** - 从用户场景出发，而非从技术出发
2. **能力导向** - 用户看到的是能力，不是算法
3. **简化架构** - 3个核心智能体，清晰易懂
4. **渐进发展** - 分阶段实现，持续交付价值

### 10.2 与传统方案的本质差异

| 维度 | 传统方案 | 空中空间智能体 |

| **思维** | 功能堆砌 | 能力组合 |

| **交互** | 命令控制 | 意图理解 |
