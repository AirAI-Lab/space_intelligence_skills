# 空中空间智能体 - 系统总览

> **核心理念**: 让无人机具备空间感知、认知和行动能力

---

## 1. 产品定位

**SkyEdge AI 是一个空中空间智能体系统**

我们构建的不是"另一个大疆妙算3"或"另一个巨视安全"，而是一个让无人机从"工具"变成"伙伴"的智能体系统。

| 传统无人机 | 空中空间智能体 |
|-----------|---------------|
| 远程控制的工具 | 自主思考的伙伴 |
| 执行预定任务 | 理解环境并决策 |
| 单机作业 | 多机协同 |
| 静态能力 | 持续学习进化 |

---

## 2. 核心架构

### 2.1 三大核心智能体

我们采用 **3+1** 的架构：3个核心智能体 + 1个云边协同平台

```
┌─────────────────────────────────────────────────────────────────┐
│                    空中空间智能体系统                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                     云边协同平台                          │   │
│  │     设备管理 │ 模型管理 │ 训练服务 │ OTA升级 │ 监控告警    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↕ MQTT/HTTP                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │              │  │              │  │              │         │
│  │  感知智能体   │  │  认知智能体   │  │  行动智能体   │         │
│  │              │  │              │  │              │         │
│  │  Perception  │  │  Cognition   │  │   Action     │         │
│  │              │  │              │  │              │         │
│  │    "看"      │  │    "想"      │  │    "做"      │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 智能体职责

| 智能体 | 核心职责 | 用户视角 | 能力模块 |
|--------|----------|----------|----------|
| **感知智能体** | 理解环境 | "无人机能看到什么" | 目标识别、变化检测、语义分割、三维重建 |
| **认知智能体** | 分析决策 | "无人机理解什么" | 空间推理、路径规划、任务分配、风险评估 |
| **行动智能体** | 执行任务 | "无人机能做什么" | 自主飞行、协同作业、人机交互 |

### 2.3 为什么是3个智能体？

**简洁的"看-想-做"逻辑**：

```
感知（看）→ 认知（想）→ 行动（做）
    ↓           ↓           ↓
  看到目标    规划路径    飞往目标
  发现变化    分析原因    执行响应
```

用户无需理解复杂的技术细节，只需要知道无人机能"看"、"想"、"做"。

## 5. 技术实现

### 5.1 技术选型原则

> **技术是为能力服务的，用户不需要关心技术细节**

| 原则 | 说明 |
|------|------|
| **能力优先** | 用户选择能力，系统选择技术 |
| **可替换性** | 技术实现可以随时升级替换 |
| **透明化** | 用户无需了解底层算法 |

### 5.2 当前技术选型

**感知智能体**：

| 能力 | 当前实现 | 备选方案 |
|------|----------|----------|
| 目标识别 | YOLOv8 | RT-DETR, YOLOv9 |
| 变化检测 | RCMT V3 | ChangeFormer |
| 语义分割 | SegFormer | UPerNet |
| 目标跟踪 | DeepSORT | ByteTrack |

**认知智能体**：

| 能力 | 当前实现 |
|------|----------|
| 路径规划 | A*算法 |
| 任务分配 | 拍卖算法 |

**基础设施**：

| 组件 | 技术 |
|------|------|
| 边缘推理 | TensorRT |
| 通信协议 | MQTT + HTTP |
| 云端后端 | Spring Boot |
| 云端前端 | Vue3 + Element Plus |

## 6. 应用场景

### 6.1 场景驱动的产品设计

**我们不是从技术出发，而是从用户场景出发**

```
用户场景 → 需要的能力 → 技术实现
    ↓
"我需要监测这片区域有没有新增建筑"
    ↓
需要: 变化检测 + 语义分割
    ↓
技术: RCMT + SegFormer
```

### 6.2 典型场景

| 场景 | 需要的能力 | 智能体组合 |
|------|------------|-----------|
| **城市违建监测** | 变化检测 + 变化分析 | 感知 + 认知 |
| **电力巡检** | 目标识别 + 缺陷分类 | 感知 + 认知 |
| **农业监测** | 变化检测 + 语义分割 | 感知 + 认知 |
| **应急救援** | 目标识别 + 路径规划 | 感知 + 认知 + 行动 |
| **多机巡检** | 协同作业 + 任务分配 | 感知 + 认知 + 行动 |

## 7. 开发团队协作

### 7.1 智能体分工

| 角色 | 负责模块 | 代码路径 |
|------|----------|----------|
| **软件开发** | 边缘框架、云边平台 | `edge_infer/src/*`, `edge_infer_cloud/backend/*` |
| **算法开发** | 模型训练、优化 | `edge_infer_cloud/models/*`, `edge_infer_cloud/training/*` |
| **前端开发** | UI/UX | `edge_infer_cloud/frontend/*` |
| **产品经理** | 需求、路线图 | 文档、规划 |

### 7.2 开发流程

```
1. 场景分析 → 产品经理
2. 能力设计 → 产品经理 + 技术负责人
3. 技术选型 → 技术负责人
4. 开发实现 → 各自负责模块
5. 集成测试 → 软件开发
6. 能力验证 → 算法开发
7. 部署上线 → 云边协同
```
