# 训练与模型交付指南

## 1. 目标

规范数据集、训练任务、模型包、评测、转换和部署流程，确保模型可追踪、可评估、可部署、可回滚。

训练模块采用“数据飞轮 + 零代码训练 + 一键导出 + 行业预设”的产品形态。普通用户通过向导完成训练闭环，算法工程师通过高级模式调整专业参数。

## 2. 数据集

数据集应包含：

- 原始样本。
- 标注文件。
- 标签定义。
- 训练集、验证集、测试集划分。
- 数据版本。
- 数据来源。

## 3. 数据飞轮

数据飞轮用于降低数据准备和标注门槛。

能力：

- 视频自动抽帧。
- 图片去重。
- 模糊、过暗、过曝图片识别。
- 类别数量统计。
- 类别不均衡检查。
- 场景覆盖度检查。
- 训练集、验证集、测试集划分检查。
- 模型辅助预标注。
- 人工复核。
- 样本回流。

数据健康度报告包含：

- 总样本数。
- 每类样本数。
- 标注完成率。
- 重复图片比例。
- 低质量图片比例。
- 类别不均衡程度。
- 缺失场景提醒。
- 推荐补充数据。
- 是否满足训练条件。

训练前必须生成数据健康度报告。数据不足时，平台允许试训，但必须标记为“试验模型”，不能直接进入生产部署。

## 4. 辅助标注

辅助标注采用“模型预标注 + 人工确认”的方式。

支持方式：

- 使用已有 YOLO 模型进行目标框预标注。
- 使用零样本或开放词汇模型进行候选目标发现。
- 使用 SAM 系列模型进行轮廓辅助标注。
- 使用历史模型对新增样本做预标注。
- 使用边缘告警样本进行快速复核。

标注结果必须经过人工确认后进入正式数据版本。

标注任务状态：

- `PENDING`
- `AUTO_LABELING`
- `REVIEWING`
- `COMPLETED`
- `REJECTED`

## 5. 零代码训练

普通用户使用零代码训练向导。

步骤：

1. 选择行业场景。
2. 选择算法能力。
3. 上传图片、导入视频或选择回流样本。
4. 完成辅助标注和人工复核。
5. 查看数据健康度。
6. 选择训练目标。
7. 点击开始训练。
8. 查看训练进度和结果。

训练目标：

- 速度优先：选择轻量模型、小输入尺寸和边缘导出配置。
- 均衡模式：选择默认模型、默认增强和标准评测。
- 精度优先：选择较大模型、较高输入尺寸和更长训练轮次。

平台自动处理：

- 基础模型选择。
- 批大小估算。
- 训练轮次建议。
- 输入尺寸建议。
- 预训练权重选择。
- 早停策略。
- 混合精度训练。
- 训练集划分。
- 训练日志和指标记录。

高级模式保留以下参数：

- `epochs`
- `batch`
- `imgsz`
- `optimizer`
- `lr0`
- `weight_decay`
- `augmentation`
- `freeze`
- `resume`

## 6. 行业预设

行业预设用于让用户从业务场景开始，而不是从模型参数开始。

预设内容：

- 场景名称。
- 默认标签。
- 标注说明。
- 推荐基础模型。
- 训练目标。
- 数据健康度阈值。
- 评测指标。
- 导出格式。
- 目标设备。
- 告警规则模板。

首批预设：

- 安全生产。
- 水利巡检。
- 施工安全。
- 交通监管。
- 环保监管。
- 园区监测。

## 7. 训练任务

训练任务记录：

- 数据集版本。
- 算法模板。
- 训练参数。
- 启动时间。
- 结束时间。
- 状态。
- 日志。
- 指标。
- 失败原因。

## 8. 训练追踪

默认追踪方式：

- 平台数据库保存任务、参数、指标和状态。
- 对象存储保存日志、模型文件、评测报告和曲线图片。
- 前端训练详情页展示训练曲线、日志、artifact 和失败原因。

MLflow 为可选增强：

- `MLFLOW_ENABLED=false` 时训练服务不调用 MLflow。
- MLflow 写入失败不影响训练任务。
- MLflow DNS 失败不影响训练任务。

## 9. 模型包规范

模型交付包包含：

- `best.pt` 或模型文件。
- `model_config.json`
- `capability_config.json`
- `data.yaml`
- `classes.txt`
- `results.csv`
- `args.yaml`
- `eval_report.json`
- `deployment_profile.json`
- `README.md`

## 10. 能力配置

算法能力采用“场景 + 基础模型 + 能力配置”方式。

能力配置包含：

- 场景。
- 基础模型。
- 类别映射。
- Prompt。
- 阈值。
- 检测区域。
- 后处理。
- 告警等级。
- 证据保存规则。
- 验收指标。

## 11. 评测

上线前必须完成：

- 测试集评测。
- 误报样例。
- 漏报样例。
- 延迟。
- 吞吐。
- 显存占用。
- 适用边界。
- 不适用边界。

## 12. 一键导出

一键导出用于把训练结果转换成可部署模型包。

首期支持：

- PyTorch `best.pt`。
- ONNX。
- TensorRT FP16。
- TensorRT INT8。
- edge_infer 模型包。

后续扩展：

- OpenVINO。
- TFLite。
- CoreML。
- RKNN。
- NCNN。
- Web 推理包。
- Android/iOS 演示 App。

导出任务记录：

- 源模型版本。
- 目标格式。
- 输入尺寸。
- 精度模式。
- 校准数据集。
- 目标设备。
- 导出日志。
- 导出产物。
- 兼容性检查结果。

TensorRT INT8 导出必须使用代表性校准数据集。

## 13. 部署

部署流程：

1. 注册模型版本。
2. 上传模型包。
3. 执行兼容性检查。
4. 转换模型格式。
5. 下发到边缘设备或云端服务。
6. 查看部署状态。
7. 绑定算法任务。
8. 监控推理结果。

部署目标：

- 云端推理服务。
- edge_infer 边缘推理框架。
- 第三方 API 调用。
- 演示环境。

## 14. 样本回流

告警有效样本、误报样本、漏报样本和人工复核结果进入样本回流池，用于后续标注、评测和再训练。

## 15. 阶段规划

- 自动生成评测报告。
