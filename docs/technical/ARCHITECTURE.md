# 技术架构文档

## 1. 总体架构

平台采用云边协同架构：

- ZLMediaKit/C++：流媒体核心。
- edge_infer/C++：边缘推理框架。
- Spring Boot/Java：平台业务、权限、审计、编排。
- Python：云端训练、云端推理、抽帧分析、零样本推理、大模型推理和评测。
- Vue3/TypeScript：管理端和空间态势工作台。
- PostgreSQL/TimescaleDB、Redis、EMQX、SeaweedFS：基础数据底座。

## 2. 模块划分

- `device`：设备、边缘节点、能力、心跳、命令、OTA。
- `video`：媒体服务、视频设备、视频通道、流代理、截图、录像。
- `spatial`：地图供应商、图层、点位、区域、轨迹、数字孪生资产。
- `algorithm`：算法模板、基础模型、能力配置、算法任务。
- `inference`：推理结果、云端复核、结果聚合。
- `alert`：告警规则、告警事件、证据、处置记录。
- `dataset`：数据集、数据版本、样本、标注、样本回流。
- `training`：训练任务、训练参数、日志、指标、产物。
- `model`：模型版本、模型文件、评测、转换、部署。
- `assistant`：AI 助手、报表、工具调用、知识库。
- `system`：用户、角色、权限、审计、Webhook、系统配置。

## 3. 视频架构

ZLMediaKit 作为独立容器部署。平台通过 HTTP API 控制 ZLMediaKit，通过 WebHook 接收流状态事件。

平台负责：

- 视频设备和通道业务数据。
- 拉流代理创建和状态管理。
- 播放 URL 生成。
- 截图和录像业务索引。
- 播放和推流鉴权。

ZLMediaKit 负责：

- 协议接入。
- 推拉流。
- 转协议。
- 截图。
- 录像。
- 流事件通知。

## 4. 边缘推理架构

`edge_infer` 是独立 C++ 边缘推理框架。

平台负责：

- 设备注册。
- 模型版本管理。
- 算法任务配置。
- 结果接收。
- 告警生成。
- 样本回流。
- 部署状态展示。

`edge_infer` 负责：

- 视频取流。
- 解码。
- 推流。
- 边缘推理。
- 离线缓存。
- 结果上报。
- 命令回执。

## 5. 训练架构

训练服务采用 Python 实现，支持数据集读取、训练、指标上报、模型文件保存和评测。

训练追踪默认写入平台数据库和对象存储。MLflow 作为可选增强，不作为训练成功的强依赖。

## 6. 空间架构

空间能力通过 `MapProvider` 和 `SpatialViewProvider` 抽象：

- `MapProvider`：天地图、离线瓦片、私有 GIS。
- `SpatialViewProvider`：CesiumJS、MapLibre GL、Three.js 或私有数字孪生引擎。

空间对象包括点、线、面、轨迹和三维资产引用。

## 7. AI 助手架构

AI 助手通过后端受控工具访问平台数据。

规则：

- 大模型不能直接访问数据库。
- 工具调用继承用户权限。
- 回答必须包含数据来源。
- 删除、重启、部署和命令下发必须人工确认。
- 所有调用写入审计日志。
