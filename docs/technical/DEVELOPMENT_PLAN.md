# 云边协同空间智能平台开发计划

## 1. 计划目标

本计划用于指导平台从当前可运行版本升级为云边协同空间智能平台。

开发目标：

- 保持当前系统可运行。
- 优先完成真实业务闭环。
- 不重写 `edge_infer` C++ 边缘推理框架。
- 不自研流媒体内核。
- 不一次性建设重型训练平台。
- 先完成 MVP，再逐步扩展三维地图、数字孪生、大模型增强和高级 AutoML。

## 2. 当前文档基础

当前正式文档可以支撑开发：

- `docs/business/`：说明市场定位、场景方案和竞争优势。
- `docs/product/PRODUCT_REQUIREMENTS.md`：定义产品功能、用户角色、MVP 范围。
- `docs/product/UI_UX_DESIGN_GUIDE.md`：定义页面、空间态势、模型工厂和大屏风格。
- `docs/technical/ARCHITECTURE.md`：定义技术边界和模块划分。
- `docs/technical/API_AND_INTEGRATION.md`：定义第三方、边缘框架、地图、大模型和 Webhook 集成边界。
- `docs/technical/TRAINING_AND_MODEL_GUIDE.md`：定义训练、数据飞轮、辅助标注、零代码训练、一键导出和模型包。
- `docs/technical/OPERATIONS_AND_TROUBLESHOOTING.md`：定义运维和故障处理。
- `README_DOCKER.md`：定义当前可运行 Docker 部署流程。

当前文档已经具备产品、技术、部署、集成和训练闭环。后续开发中需要持续补充接口字段、数据库迁移、页面原型和验收用例。

## 3. 开发阶段

### 阶段 0：稳定基线

周期：1-2 周

目标：保持现有平台稳定运行，为后续改造建立基线。

后端任务：

- 统一任务状态枚举。
- 梳理现有设备、数据集、训练、模型、转换、部署、推理、告警接口。
- 明确 MLflow 可选化策略。
- 补充核心接口错误码。
- 补充健康检查。

前端任务：

- 梳理现有路由。
- 标记保留、合并、下沉和删除页面。
- 建立统一布局、状态、表格、详情抽屉和任务进度组件。

训练服务任务：

- 确认 `MLFLOW_ENABLED=false` 默认生效。
- 训练指标写入平台数据库。
- 训练日志和模型文件写入对象存储。
- 训练失败不因 MLflow 连接失败中断。

验收：

- 当前 Docker 部署可启动。
- 设备、训练、模型、部署、推理、告警基础链路可用。
- 训练失败能看到明确错误原因。

## 4. 阶段 1：空间态势与视频接入 MVP

周期：2-3 周

目标：建立空间智能平台的第一入口，把摄像头、边缘设备和视频通道放到地图上。

后端任务：

- 新增 `spatial` 模块。
- 新增地图供应商配置。
- 新增地图图层配置。
- 新增区域、点位、设备位置、通道位置接口。
- 新增 `video` 模块基础表和接口。
- 对接当前 RTMP 演示链路。
- 预留 ZLMediaKit 接口适配层。

前端任务：

- 新增空间态势页面。
- 接入天地图或离线瓦片。
- 实现设备点位、摄像头点位、告警点位图层。
- 实现点位弹窗。
- 实现点位联动视频预览。
- 新增视频接入页面。

部署任务：

- 保持当前 nginx-rtmp 可运行。
- 新增 ZLMediaKit compose 草案和配置说明。
- 不在本阶段强制替换当前 RTMP 服务。

验收：

- 地图可显示。
- 摄像头和边缘设备可绑定点位。
- 点击摄像头点位可打开视频预览。
- 视频通道状态可展示。

## 5. 阶段 2：边缘推理结果接入与告警闭环

周期：2-3 周

目标：完成 `edge_infer` 结果接入、告警生成、证据展示和处置闭环。

后端任务：

- 规范 `edge_infer` MQTT/HTTP 上报字段。
- 推理结果绑定设备、通道、任务、模型版本和空间位置。
- 扩展告警事件和告警证据模型。
- 支持告警有效、误报、已处理状态。
- 支持样本回流入口。

前端任务：

- 新增推理结果列表。
- 新增告警详情页面。
- 告警详情展示图片、视频片段、地图位置、设备、通道、模型和任务。
- 地图点位联动告警详情。

edge_infer 对接任务：

- 确认上报 topic。
- 确认结果 JSON schema。
- 确认证据上传方式。
- 确认命令回执格式。

验收：

- `edge_infer` 可上报推理结果。
- 平台可生成告警。
- 告警可在地图定位。
- 用户可完成告警处置和样本反馈。

## 6. 阶段 3：模型工厂 MVP

周期：3-5 周

目标：让普通用户通过向导完成数据准备、辅助标注、零代码训练、评测和模型导出。

后端任务：

- 新增数据版本。
- 新增样本文件和标注记录。
- 新增数据健康度报告。
- 新增标注任务状态。
- 新增行业预设模板。
- 新增训练向导接口。
- 新增模型包导出任务。

训练服务任务：

- 支持视频抽帧。
- 支持图片去重。
- 支持低质量图片检测。
- 支持 YOLO 预标注。
- 支持零代码训练参数生成。
- 支持 ONNX 导出。
- 支持 TensorRT FP16 导出。
- 预留 TensorRT INT8 校准数据接口。
- 生成 `edge_infer` 模型包。

前端任务：

- 模型工厂向导。
- 数据上传和视频抽帧页面。
- 标注复核页面。
- 数据健康度页面。
- 训练目标选择。
- 训练进度和日志页面。
- 评测结果页面。
- 模型包导出页面。

验收：

- 用户可上传数据或导入视频抽帧。
- 平台可生成数据健康度报告。
- 平台可进行 YOLO 预标注和人工复核。
- 用户可选择速度优先、均衡模式、精度优先启动训练。
- 训练完成后可导出 ONNX/TensorRT/edge_infer 模型包。

## 7. 阶段 4：AI 助手与报表 MVP

周期：2-3 周

目标：提供平台问答、告警汇总、训练进度查询和日报/月报草稿。

后端任务：

- 新增大模型供应商配置。
- 新增 AI 助手会话和消息记录。
- 新增工具调用审计。
- 新增只读工具接口。
- 新增日报/月报生成任务。

前端任务：

- 新增 AI 助手抽屉。
- 支持页面上下文提问。
- 支持日报/月报草稿查看和导出。
- 回答中对象可点击跳转。

安全任务：

- 工具调用继承用户权限。
- 默认脱敏。
- 不允许 AI 助手直接执行删除、重启、部署和命令下发。

验收：

- 用户可查询今日告警。
- 用户可查询训练任务进度。
- 用户可查询设备和视频通道状态。
- 用户可生成日报和月报草稿。
- AI 回答包含数据来源。

## 8. 阶段 5：ZLMediaKit 正式替换与视频增强

周期：2-4 周

目标：将当前演示 RTMP 链路升级为 ZLMediaKit 视频底座。

后端任务：

- 封装 ZLMediaKit HTTP API。
- 接收 ZLMediaKit WebHook。
- 支持 RTSP 拉流代理。
- 支持截图。
- 支持录像任务。
- 支持播放鉴权。

前端任务：

- 视频通道状态实时更新。
- 支持 WebRTC、HTTP-FLV、HLS 播放地址。
- 支持录像检索和回放入口。

部署任务：

- 将 ZLMediaKit 加入 Docker Compose。
- 保留 nginx-rtmp 兼容或迁移说明。
- 更新 README_DOCKER.md。

验收：

- RTSP 摄像头可接入。
- 前端可稳定预览。
- 流上线和下线状态可同步。
- 平台可截图和录像。

## 9. 阶段 6：空间智能增强与数字孪生预留

周期：3-6 周

目标：增强空间态势能力，预留三维地图和数字孪生大屏。

后端任务：

- 空间对象扩展点、线、面和三维资产引用。
- 支持轨迹上报和回放。
- 支持告警热力图。
- 支持地图框选查询。
- 支持数字孪生场景配置。

前端任务：

- 图层面板。
- 区域绘制。
- 轨迹回放。
- 告警热力图。
- 地图框选统计。
- 大屏模式。
- 预留 Cesium/MapLibre/Cesium 入口。

验收：

- 用户可按地图区域筛选设备、通道和告警。
- 用户可查看移动设备轨迹。
- 用户可查看告警热力图。
- 大屏可展示项目态势。

## 10. 阶段 7：生产化增强

周期：4-8 周

目标：让平台满足真实项目长期运行。

任务：

- 用户、角色、项目权限。
- API Key 管理。
- Webhook 签名。
- 操作审计。
- 备份恢复。
- 服务监控。
- 日志检索。
- 部署脚本完善。
- 文档和验收用例完善。

验收：

- 支持生产环境部署。
- 支持故障定位。
- 支持备份恢复。
- 支持权限隔离。
- 支持客户试运行。

## 11. 版本里程碑

周期：8-12 周

包含：

- 空间态势。
- 视频接入基础。
- 边缘结果接入。
- 告警闭环。
- 模型工厂 MVP。
- AI 助手 MVP。

周期：3-5 个月

包含：

- 权限和审计。
- 运维监控。
- 地图态势增强。

### 11.3 规模化版本

周期：6-9 个月

包含：

- 10 大场景。
- 100+ 能力项。
- 三维地图。
- 数字孪生大屏。
- 高级 AutoML。
- 多项目交付能力。

## 12. 当前不做

- 不重写 `edge_infer`。
- 不自研流媒体内核。
- 不一开始接入完整 K8s 训练平台。
- 不一开始生成 Android/iOS App。
- 不一开始做全量自动超参数搜索。
- 不一次性覆盖 100+ 算法能力。
