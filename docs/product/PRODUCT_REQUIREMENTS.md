# 产品需求文档

## 1. 产品目标

建设云边协同空间智能平台，实现设备接入、视频流管理、边缘推理结果接入、云端训练、大模型分析、空间态势、告警处置和报表闭环。

## 2. 用户角色

- 客户管理员：查看项目运行状态、设备在线率、告警统计和报表。
- 运维人员：管理设备、视频通道、服务健康、日志和故障。
- 算法人员：管理数据集、训练任务、模型版本、评测和模型包。
- 业务人员：查看地图态势、实时视频、告警事件和日报/月报。
- 系统管理员：管理用户、角色、权限、Webhook、地图供应商和大模型供应商。

## 3. 主导航

- 总览。
- 空间态势。
- 视频接入。
- 智能分析。
- 告警处置。

### 4.7 模型工厂

- 数据集。
- 视频抽帧。
- 数据健康度分析。
- 模型辅助标注。
- 人工复核。
- 数据版本。
- 训练任务。
- 零代码训练向导。
- 行业预设模板。
- 训练日志。
- 训练指标。
- 模型版本。
- 模型评测。
- 模型转换。
- 模型部署。

模型工厂面向两类用户：

- 普通业务用户：通过向导完成数据上传、辅助标注、自动训练、评测和部署。
- 算法工程师：进入高级模式配置训练参数、模型结构、增强策略和导出参数。

模型工厂默认隐藏学习率、优化器、损失函数等专业参数，提供三种训练目标：

- 速度优先：面向边缘设备和低延迟场景。
- 均衡模式：面向常规项目交付。
- 精度优先：面向服务器推理和高准确率场景。

模型工厂提供行业预设：

- 安全生产。
- 水利巡检。
- 施工安全。
- 交通监管。
- 环保监管。
- 园区监测。

每个行业预设包含默认标签、基础模型、标注规范、训练模板、评测指标、导出格式和部署目标。

- 月报生成。

## 5. MVP 范围

- 二维地图和点位绑定。
- RTSP 视频接入和预览。
- 边缘推理结果接入。
- 告警事件和证据展示。
- 数据集、训练任务和模型版本基础闭环。
- 数据健康度分析。
- 模型辅助预标注。
- 零代码训练向导。
- ONNX/TensorRT 模型包导出。
- AI 助手只读问答和日报/月报草稿。

## 6. 不包含内容

- 不自研流媒体内核。
- 不重写 C++ 边缘推理框架 `edge_infer`。
- 不一次性建设重型 K8s 训练平台。
- 不一次性覆盖全部 100+ 算法能力。
- 不允许 AI 助手绕过权限直接操作数据库或设备。
