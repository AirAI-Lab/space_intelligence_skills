# UI/UX 设计规范

## 1. 设计目标

平台 UI 面向空间智能业务，采用“专业运维控制台 + 地图态势工作台 + 数字孪生大屏”的设计方向。

## 2. 设计原则

- 好用：关键任务路径短，状态清楚。
- 易用：复杂配置分步引导。
- 稳定：异步任务有进度、日志和失败原因。
- 专业：视觉克制，信息密度高。
- 空间化：设备、视频和告警优先与地图点位关联。

## 3. 视觉风格

- 日常管理采用浅色后台风格。
- 视频监控、地图态势和大屏采用深色模式。
- 主色采用科技蓝。
- 在线使用绿色，预警使用橙色，告警使用红色，离线使用灰色。
- 避免大面积装饰渐变、光斑和营销式视觉。
- 卡片圆角控制在 6-8px。
- 表格、筛选和操作按钮保持紧凑。

## 4. 页面布局

### 4.1 总览

- 顶部指标卡。
- 中部趋势图。
- 右侧高优先级告警。
- 底部服务健康和快捷操作。

### 4.2 空间态势

- 全屏地图。
- 左侧区域树和图层面板。
- 右侧点位、设备、视频、告警详情。
- 底部事件时间轴。
- 支持地图框选、点位聚合、热力图和轨迹。

### 4.3 视频接入

- 左侧设备和通道树。
- 中间视频宫格。
- 右侧流状态、截图、录像和通道操作。

### 4.4 智能分析

- 任务列表。
- 状态筛选。
- 任务详情抽屉。
- 检测区域编辑器。

### 4.5 告警处置

- 左侧告警列表。
- 中间证据预览。
- 右侧处置时间线。
- 底部样本反馈。

### 4.6 模型工厂

- 按数据集、训练、评测、转换、部署组织流程。
- 每个任务显示状态、进度、日志、产物和失败原因。
- 普通用户使用向导模式。
- 算法工程师使用高级模式。
- 向导模式隐藏学习率、优化器、损失函数等专业参数。
- 训练目标使用“速度优先、均衡模式、精度优先”表达。
- 数据健康度用分数、风险项和补数建议表达。
- 标注页面提供模型预标注、人工修正和批量确认。
- 评测页面展示是否可部署，而不是只展示指标数字。

模型工厂向导步骤：

1. 选择行业场景。
2. 导入数据。
3. 辅助标注。
4. 数据健康检查。
5. 选择训练目标。
6. 开始训练。
7. 查看评测。
8. 导出模型包。
9. 部署到云端或边缘。

### 4.7 AI 助手

- 全局抽屉入口。
- 支持页面上下文提问。
- 支持报表草稿编辑。
- 回答中对象可点击跳转。

## 5. 组件规范

- `StatusBadge`：统一状态展示。
- `MetricCard`：统一指标卡。
- `HealthIndicator`：统一服务健康。
- `TaskProgress`：统一任务进度。
- `EvidenceViewer`：统一图片、视频和 JSON 证据。
- `VideoGrid`：统一视频宫格。
- `RegionEditor`：统一检测区域绘制。
- `TimelineLog`：统一时间线。
- `AssistantDrawer`：统一 AI 助手。
- `MapLayerPanel`：统一地图图层。
- `SpatialObjectPopup`：统一地图点位弹窗。

## 6. 数字孪生大屏

大屏用于汇报、展示和指挥调度。

设计要求：

- 三维或地图场景居中。
- 两侧指标面板。
- 底部事件时间轴。
- 告警突出显示。
- 轨迹、热力、设备、视频图层可切换。
- 大屏只展示和联动，不承载复杂配置。

## 7. 首批重构页面

1. 总览。
2. 空间态势。
3. 视频接入。
4. 告警处置。
5. 智能分析。
6. 模型工厂。
7. AI 助手。
