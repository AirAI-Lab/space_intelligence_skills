# 空中智能体 - 业务部门需求分析

> **版本**: V3.0（业务导向）
> **日期**: 2026-03-09
> **原则**: 需求优先、业务导向、场景驱动

---

## 🎯 核心问题

**当前问题**：
- ❌ 从技术角度设计（边缘/云端）
- ❌ 忽略了业务部门的实际需求
- ❌ 不同部门不会采用相同算法包
- ❌ 未考虑各部门的预算和采购方式

**正确思路**：
- ✅ 从业务场景出发
- ✅ 按部门需求分组
- ✅ 技术方案服务于业务需求
- ✅ 考虑采购预算和决策链

### 一、应急管理部门（应急管理厅/局）

#### 1.1 核心职责
- 自然灾害监测预警
- 安全生产事故应急
- 城市安全风险监测
- 应急救援指挥

#### 1.2 业务场景

**场景1：灾害监测预警**
```
需求：实时监测自然灾害风险
- 山体滑坡风险区域
- 泥石流易发区
- 洪涝灾害监测
- 森林火灾预警

使用频率：7×24小时实时监测
实时性要求：高（<1分钟）
预算来源：应急管理专项资金
采购特点：政府采购，年度预算
```

**场景2：应急救援**
```
需求：灾后快速评估和救援
- 受灾面积评估
- 道路阻断检测
- 救援目标定位
- 人员搜救辅助

使用频率：灾害发生时
实时性要求：极高（<30秒）
预算来源：应急专项资金
采购特点：紧急采购，成本不敏感
```

**场景3：城市安全监测**
```
需求：城市重点区域安全监测
- 人员密集场所监控
- 危险源监测

使用频率：日常监测
```

## 📝 总结

### 核心结论

1. ✅ **业务导向**：从业务部门需求出发，而非技术角度
2. ✅ **差异化服务**：每个部门定制算法包
3. ✅ **灵活定价**：按部门/场景/规模差异化定价
4. ✅ **技术适配**：边缘/云端根据业务需求选择

### 关键差异

| 维度 | 原方案（技术导向） | 新方案（业务导向） |
|------|------------------|------------------|
| **分组依据** | 技术特点（实时/周期） | 业务部门（应急/住建/国土...） |
| **算法包** | 通用包（20/30/50/70种） | 部门定制包（8-15种） |
| **定价模式** | 统一定价 | 差异化定价 |
| **客户理解** | 难理解（技术术语） | 易理解（业务术语） |
| **销售难度** | 高（需技术解释） | 低（直接对话业务） |
