# 模型部署指南

本指南介绍如何将训练好的模型部署到边缘设备。

## 部署流程概览

```
训练完成 → 导出模型 → 上传模型 → 选择设备 → 一键部署 → 验证部署
```

## 第一步：上传模型

### 通过Web界面

1. 点击"模型管理" → "上传模型"
2. 填写模型信息：

**基础信息**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 模型名称 | 模型显示名称 | 安全帽检测v2.0 |
| 模型类型 | 模型类型 | yolov8 |
| 模型格式 | 文件格式 | TensorRT Engine |
| 模型文件 | 上传文件 | best.engine |
| 配置文件 | 模型配置 | model_config.yaml |

**配置文件示例**：

```yaml
# model_config.yaml
model:
  name: "helmet_detect_v2"
  format: "tensorrt"
  input:
    width: 960
    height: 960
    channels: 3
  output:
    num_classes: 2
    names: ["person", "helmet"]
  threshold:
    confidence: 0.5
    nms: 0.45
```

3. 点击"上传"

### 通过API

```bash
curl -X POST http://localhost:8080/api/v1/model/upload \
  -F "model_file=@best.engine" \
  -F "config_file=@model_config.yaml" \
  -F "name=helmet_detect_v2" \
  -F "type=yolov8" \
  -F "format=tensorrt"
```

### 通过CLI

```bash
edge-cli model upload \
    --file best.engine \
    --config model_config.yaml \
    --name "安全帽检测v2.0" \
    --type yolov8
```

## 第三步：选择目标设备

### 设备筛选

1. 在模型详情页，点击"部署"
2. 进入设备选择页面

筛选条件：

| 条件 | 选项 |
|------|------|
| 设备类型 | Jetson Xavier/Orin/Nano |
| 设备状态 | 在线/离线 |
| 设备分组 | 巡检组A/巡检组B |

### 设备兼容性检查

系统自动检查：

- 硬件兼容性（GPU算力、显存）
- 软件兼容性（TensorRT版本）
- 资源可用性（存储空间）

**兼容性示例**：

| 设备 | 兼容性 | 说明 |
|------|--------|------|
| EDGE_001 | ✅ 兼容 | 满足所有要求 |
| EDGE_002 | ⚠️ 警告 | 显存不足，需要优化 |
| EDGE_003 | ❌ 不兼容 | TensorRT版本过低 |

### 批量部署

选择多个设备进行批量部署：

1. 勾选目标设备
2. 点击"批量部署"
3. 确认部署信息

## 第四步：一键部署

### 部署配置

**部署参数**：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 部署策略 | 立即/定时 | 立即 |
| 备份当前模型 | 是/否 | 是 |
| 自动重启推理 | 是/否 | 是 |
| 验证部署 | 是/否 | 是 |

### 开始部署

1. 确认部署配置
2. 点击"开始部署"
3. 系统自动执行：

   - 备份当前模型
   - 加载新模型
   - 运行验证测试
   - 切换到新模型

## 第五步：验证部署

**验证项目**：
- ✅ 模型加载成功

## 部署管理

### 部署回滚

如果新模型有问题，可以快速回滚：

1. 进入"部署记录"
2. 选择之前的版本
3. 点击"回滚"
4. 系统自动恢复

### A/B测试

同时部署两个模型进行对比：

1. 选择两个模型版本
2. 分配流量比例（50%:50%）
3. 运行一段时间
4. 对比性能指标
5. 选择更优版本

### 部署策略

#### 灰度发布

逐步扩大部署范围：

```
第1天：部署到1台设备
第3天：部署到5台设备
第7天：部署到所有设备
```

#### 蓝绿部署

零停机部署：

1. 准备蓝色环境（当前版本）
2. 部署绿色环境（新版本）
3. 验证绿色环境
4. 切换流量到绿色
5. 保留蓝色环境用于回滚

## 热更新机制

### 无缝更新

edge_infer 支持运行时模型热更新：

- **无需重启**：推理服务不中断
- **平滑切换**：新旧模型平滑过渡
- **自动回滚**：失败自动恢复

### 更新流程

```
1. 下载新模型（后台）
2. 验证模型（后台）
3. 加载模型（后台）
4. 切换推理（无中断）
5. 卸载旧模型（后台）
```

### 配置热更新

除了模型，配置也支持热更新：

```json
{
  "model_config": {
    "conf_threshold": 0.6,
    "nms_threshold": 0.45
  },
  "inference_config": {
    "max_fps": 30
  }
}
```

下发配置后立即生效。

## 常见问题

### Q1: 部署失败怎么办？

1. 检查设备兼容性
2. 检查网络连接
3. 查看设备日志
4. 尝试回滚

### Q2: 如何批量部署？

选择设备分组，点击"批量部署"即可。

### Q3: 部署后性能下降怎么办？

1. 检查模型格式（ONNX→TensorRT）
2. 启用量化（FP16/INT8）
3. 优化输入尺寸
4. 联系技术支持

### Q4: 如何保留多个模型版本？

系统自动保留所有版本，可随时切换。
